# AmoPoint test task

Это тестовое задание для компании AmoPoint.

Техническое задание: [tz.docx](./tz.docx)

## 1. Сбор персонажей из API

Что сделано:

- консольная команда `fetch:character` получает следующего персонажа из Rick and Morty API
- API: [rickandmortyapi.com](https://rickandmortyapi.com/)
- запуск по расписанию уже настроен каждые 5 минут в `bootstrap/app.php`
- `GET /api/characters` отдает список персонажей в формате JSON

Как проверить:

```bash
php artisan fetch:character
```

Если нужно запустить расписание локально:

```bash
php artisan schedule:work
```

## 2. Скрипт для фильтрации полей формы

Что сделано:

## 3. Счетчик посещений и статистика

Что сделано:

- клиентский скрипт отправляет данные о посещениях на `POST /api/visits`
- бэкенд сохраняет события в базе
- страница `/statistics` показывает график визитов по часам и круговую диаграмму по городам
- страница статистики защищена авторизацией

Как проверить:

1. Открыть любой публичный раздел сайта, чтобы скрипт собрал событие посещения
2. Авторизоваться
3. Открыть `/statistics`

Логин для проверки создается командой:

```bash
php artisan admin:create
```
